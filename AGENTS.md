# Agent Guide: Blacklight Spotlight (`blacklight-spotlight`)

Spotlight is a Rails engine that extends [Blacklight](https://github.com/projectblacklight/blacklight) for digital exhibits. Everything is namespaced under `Spotlight::` (engine uses `isolate_namespace`).

- Branches: `main` is the 5.x series; `release-4.x` holds 4.x backports.
- Ruby: gemspec requires `>= 3.1`, RuboCop `TargetRubyVersion` is 3.1 (code must target 3.1). CI runs Ruby 3.3 / 3.4 / 4.0 against Rails 7.2 and 8.1.

## Commands

### One-command test run (preferred)
```bash
bundle exec rake      # = rubocop + ci: starts Solr, loads fixtures, runs spec:prepare, runs rspec
```
The default task (Rakefile) runs RuboCop **then** `ci`, which auto-starts Solr (via `solr_wrapper`, or Docker if available) and tears it down. Do **not** start Solr yourself before running bare `rake`.

### Running a single spec / subset (needs Solr already up)
`rspec` does **not** bootstrap anything — Solr must be running and fixtures prepared first:
```bash
docker compose up -d                       # or: solr_wrapper   (separate tab)
rake spotlight:fixtures                    # seeds Solr; run once after (re)generate
cd .internal_test_app && rake spec:prepare && cd -   # generates CSS assets tests need; once
bundle exec rspec spec/models/exhibit_spec.rb:42     # file[:line]
```
`spec:prepare` builds CSS used by feature specs; clear with `rake assets:clobber`.

### Lint
```bash
bundle exec rubocop          # single quotes enforced; Layout/LineLength max 160
bundle exec i18n-tasks health  # translation health check
```

### Internal test app (`.internal_test_app/`, git-ignored, generated by `engine_cart`)
- **Never run root-level `rails`/`db:` commands** — they target the wrong context. Anything DB-related runs inside `.internal_test_app/`.
- Regenerate when stale: `rake engine_cart:regenerate` (clean+generate). Other tasks: `rake engine_cart:generate`, `rake engine_cart:clean`.
- `ActiveRecord::PendingMigrationError`? → `cd .internal_test_app && RAILS_ENV=test bundle exec rails db:schema:load`.

### Dev server
```bash
rake spotlight:server        # builds test app, starts Solr (solr_wrapper), runs rails
```

## JavaScript — generated bundle, do not hand-edit
- Source lives in `app/javascript/spotlight/`; entrypoint is `index.js` (imports `core.js` plus the `admin/`, `user/`, `controllers/` directories).
- `npm run prepare` (rollup) builds **generated** bundles into `app/assets/javascripts/spotlight/spotlight.js{,.esm.js}`. Never edit those by hand — changes are overwritten.
- Prefer modern **vanilla ES6+** for new/refactored code; avoid new jQuery and migrate existing usages where practical.

## Architecture essentials (non-obvious)
- **Exhibits (`Spotlight::Exhibit`)** are the top-level container; everything is exhibit-scoped.
- **Pages** (`Spotlight::Page` / `HomePage` / `FeaturePage` / `AboutPage`) hold curated content via Sir Trevor blocks (`Spotlight::Page` `content` is Sir Trevor JSON).
- **`Spotlight::SolrDocumentSidecar`** stores exhibit-specific metadata in the DB alongside Solr docs — this is how per-exhibit overrides persist.
- **`Spotlight::CustomField`** = admin-defined metadata fields; **`Spotlight::Resource`** = ingest source (CSV upload, IIIF, etc.) that becomes Solr docs.
- Authorization via `cancancan`; auth via `devise` + `devise_invitable`; paper_trail for versioning.

## Test suite specifics
- RSpec + Capybara. JS/feature specs run headless Chrome (`selenium_chrome_headless`) — **Chrome must be installed**.
- WebMock blocks all net except localhost (`allow_localhost: true`); stub external HTTP in specs.
- Factories: `spec/factories/` (loaded via `factory_bot`). Transactional fixtures on.
- Supports: `spec/support/**/*.rb` auto-required; feature specs get Warden + `TestFeaturesHelpers`.

## Release (only if JS changed)
Version lives in `lib/spotlight/version.rb` **and** `package.json` (keep in sync):
1. Bump both version files.
2. `npm run prepare` → `npm i --package-lock-only` → commit.
3. `bundle exec rake release` (Rubygems) → `npm publish` (npm package `spotlight-frontend`).
